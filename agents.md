# Code Quality Guidelines

Always write **high quality code**.

## Key Principles

- **Clear and readable**: Code should be self-documenting with descriptive names
- **Maintainable**: Follow DRY principles, extract common logic, avoid duplication
- **Error handling**: Handle errors gracefully with specific exceptions and meaningful messages
- **Type safety**: Use type hints and validate inputs
- **Security**: Never trust user input, use parameterized queries, manage secrets properly
- **Testing**: Write tests for critical paths and edge cases
- **Documentation**: Add docstrings and update relevant documentation

## Best Practices

- Make minimal, focused changes
- Follow existing code patterns in the codebase
- Add tests for new functionality
- Consider scalability, performance and security implications
- Review for edge cases and error scenarios
